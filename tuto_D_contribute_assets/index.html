如何升级内容

我们的内容驻留在单独的 Git LFS 仓库 中（ 基于 原始Carla资产仓库 ）。作为构建系统的一部分，我们生成并上传一个包，其中包含带有当前日期和提交标记的该内容的最新版本。我们定期升级 Carla 仓库，并提供最新版本内容包的链接。本文档包含将此链接更新到最新版本所需的手动步骤。

  1. 复制您要链接的内容包的标签。
    可以通过查看最新 Jenkins 构建 （Carla 构建 ）的工件部分中生成的包名称来找到此标签，例如 20190617_086f97f.tar.gz。

  2. 将标记粘贴到 ContentVersions.txt 中。
    通过将标记粘贴到文件末尾来 编辑 Edit ContentVersions.txt ， 例如 Latest: 20190617_086f97f (不包括 .tar.gz 部分）。或者下载 百度网盘链接 中的 software/carla/dep/Content.zip

  3. 打开拉取请求。
    提交更改并打开新的拉取请求。

!!! 注意： 如果在安装 Git LFS 前执行了 clone，则拉取下来的仓库中并不包含大文件本体，取而代之的是指向其 LFS 存储对象的文件指针，此种情况下，可通过在本地仓库中执行git lfs pull命令来拉取文件指针所指向的完整对象。
